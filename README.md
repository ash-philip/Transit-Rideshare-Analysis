# Transit Rideshare Analytics Pipeline

An end-to-end analytics pipeline built with synthetic transit rideshare data to analyze ridership, revenue, operating costs, farebox recovery, forecasting, and fare scenario impacts.

## Project Overview
This project simulates a public transit rideshare/vanpool analytics workflow using fully synthetic data. It is designed as a portfolio-safe version of a real-world business problem.

## Business Problem
A transit agency wants to evaluate and improve the performance of its rideshare service by understanding:
- ridership trends
- revenue performance
- operating costs
- farebox recovery
- the impact of fare policy changes

## Project Goals
- Generate realistic synthetic monthly transit operations data
- Build a clean ETL and analytics pipeline
- Create a master KPI dataset
- Perform descriptive and diagnostic analytics
- Forecast future ridership and revenue
- Evaluate pricing scenarios

## Tech Stack
- Python
- SQL
- pandas
- Jupyter Notebook
- statsmodels
- Git / GitHub

## Data Note
All data in this repository is synthetic and generated for portfolio purposes. No confidential or proprietary client data is included.
